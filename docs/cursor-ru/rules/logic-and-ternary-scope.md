# Правило Логики И Тернарного Оператора

- Предпочитайте явные блоки `if/else` для ветвления, которое занимает несколько строк.
- Используйте тернарный оператор только для коротких однострочных выражений.
- Не используйте вложенные тернарные операторы.
- Не используйте тернарный оператор для задания многострочных функциональных блоков или пайплайнов задач.
- В build-скриптах и конфигурационных файлах приоритет у читаемости, а не у компактной записи.

## Короткие примеры

- Хорошо: `const state = isReady ? "ready" : "idle";`
- Хорошо: `if (items.length > 0) { run(); } else { done(); }`
- Избегать: `const x = condition ? longMultilineBlock() : otherMultilineBlock();`
