# Scope задачи и согласование с пользователем

## По умолчанию

- Выполняй **только** то, что пользователь запросил в текущем сообщении, в утверждённом плане или в согласованном файле задачи (например `TASK-NN` с явным scope).
- **Не делай** drive-by рефакторинг, лишние страницы, правки toolchain, `.cursor`, коммиты и «заодно» исправления, если они **не обязательны** для завершения согласованного scope.

## Перед действием вне scope

Если собираешься сделать что-то **помимо** согласованного промпта, плана или задачи — в том числе полезное, но не запрошенное — **остановись**, сформулируй короткое **предложение** (что, зачем, какие файлы) и дождись **явного согласия** пользователя. Несогласованные действия нежелательны.

## Что не считается «лишним»

- Обязательные гейты и чеклисты для **текущей согласованной задачи на реализацию** (например `run-layout-task`, `npm run qa` при изменении HTML) — это не расширение scope.
- Read-only исследование, разумно необходимое для выполнения запроса, допустимо; читай минимально нужный объём.

## При неясности

- Если scope неоднозначен — **сначала спроси**, потом пиши и меняй файлы.
- Если регламенты (rules/commands в `.cursor`) как будто требуют работы, которую пользователь не запрашивал, **не расширяй scope молча** — согласуй с пользователем или зафиксируй `not_applicable` с причиной.
