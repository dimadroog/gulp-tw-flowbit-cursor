# new-page

Создай каркас новой страницы в рамках текущих проектных конвенций.

## Inputs

- Цель страницы и маршрут
- Ключевые секции из макета
- Требования по SEO/meta

## Steps

1. Создай шаблон страницы с обязательными семантическими landmarks (`header`, `nav`, `main`, `aside`, `footer`) и одним page-level `h1`.
2. Задай дефолтную оболочку документа: `<html lang="ru">` (если явно не нужен другой язык) и `<meta name="viewport" content="width=device-width, initial-scale=1">`.
3. Добавь минимальную head/meta-структуру, соответствующую назначению страницы, включая заглушки, если SEO-тексты неизвестны:
   - `<meta name="description" content="description">`
   - `<meta name="keywords" content="keywords">`
4. Установи `<title>`, совпадающий с названием страницы в реестре `app/index.html`.
5. Если по макету отсутствует видимый `h1`, добавь visually-hidden, но доступный `h1`.
6. Используй `section`/`article` только при наличии содержательных внутренних заголовков (`h2`/`h3`, где уместно).
7. Раздели секции на includes/partials вместо больших inline-блоков.
8. Запланируй слоты изображений по [`rules/image-delivery-and-optimization.RULE.md`](../../.cursor/rules/image-delivery-and-optimization.RULE.md) (`loading`, `picture`/`srcset`, sizing 2×, форматы).
9. Для каждого `img` зафиксируй `alt` по [`rules/accessibility-and-w3c.RULE.md`](../../.cursor/rules/accessibility-and-w3c.RULE.md).
10. Запланируй стратегию шрифтов: self-hosted доставка (если не используются только системные шрифты), форматы `WOFF/WOFF2`, только необходимые семейства/начертания.
11. Проверь, что в кастомных объявлениях шрифтов используется `font-display: swap`.
12. Выполни проверку контентоустойчивости (длинный текст, разные пропорции изображений) по image-delivery rule.
13. Зафиксируй допущения для последующих a11y и interaction-проверок.

## Done

- Каркас новой страницы готов к реализации секций.
- Повторяющиеся блоки подготовлены для выноса в loops/macros.
- Семантические landmarks и иерархия заголовков явно валидны.
- Базовые параметры документа валидны (`lang`, `viewport`, заглушки meta-тегов).
- Базовая устойчивость верстки подтверждена для вариативной длины текста и пропорций изображений.
- Стратегия изображений задокументирована по `image-delivery-and-optimization.RULE.md`; `alt` — по accessibility rule.
- Стратегия шрифтов задокументирована (форматы, хостинг, используемые начертания, `font-display`).
- `<title>` совпадает с названием страницы в `app/index.html`.
