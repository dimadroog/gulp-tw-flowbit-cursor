# validate-figma-assets

Запусти целевую post-check проверку целостности ассетов после Figma-driven реализации.

## Когда запускать

- После реализации страницы/секции по Figma.
- После изменений иконок, логотипов, аватаров или иллюстраций.
- Перед финальной сдачей, когда критична точность источника/типа ассетов.

## Проверки

1. Убедись, что векторные ассеты вставлены в шаблоны как inline SVG там, где это применимо.
2. Убедись, что растровые ассеты лежат в структурированных локальных путях внутри `app/img/`.
3. Убедись, что во шаблонах не осталось временных удаленных URL ассетов (включая Figma MCP asset URLs).
4. Убедись, что графика не подменена текстом/эмоджи-заглушками, если в дизайне это не эмоджи.
5. Убедись, что целостность вектора не нарушена (`viewBox` и геометрия path сохранены, нет деформации из-за некорректного масштабирования).
6. Убедись, что для задачи не было **незаявленного** массового переписывания исходников SVG (если файлы менялись пачкой — укажи это в evidence).
7. Убедись, что UI-SVG не принуждают к неравномерному масштабу: выполни `rg 'preserveAspectRatio="none"' app/img` и требуй **ноль совпадений**, если задача не документирует допустимое исключение по конкретным файлам.
8. После добавления или замены нескольких Figma-SVG в `app/img/layout-shell/` запусти `npm run normalize:svg-layout`, чтобы включённые в страницу иконки не дублировали `id` в одном HTML-документе.
9. Убедись, что источники изображений валидны:
   - нет пустых `src`
   - нет неразрешенных относительных путей
   - нет битых ссылок в собранном output
   - intrinsic-размеры растра соответствуют [`rules/image-delivery-and-optimization.RULE.md`](../../.cursor/rules/image-delivery-and-optimization.RULE.md) (≤ rendered × 2.0 по каждой оси)

## Результат

- Компактная validation-заметка:
  - pass/fail статус
  - нарушенные проверки (если есть)
  - точные пути, требующие исправления

## Blocking status

- Если хотя бы одна проверка не пройдена, задачу нельзя считать завершенной.
