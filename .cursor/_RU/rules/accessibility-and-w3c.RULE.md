# Политика accessibility и W3C

- Соблюдать семантические landmarks и корректную иерархию заголовков.
- Использовать семантические landmarks (`header`, `nav`, `main`, `aside`, `footer`) как обязательный baseline страницы.
- Осторожно использовать `section` и `article`: каждый такой блок должен содержать содержательный заголовок (обычно `h2` или `h3`).
- На каждой странице обязательно должен быть один `h1`. Если по макету нет видимого заголовка, добавляй доступный visually-hidden `h1`.
- Каждый `img` должен иметь атрибут `alt`: осмысленное описание для информативных изображений и пустой `alt=""` для декоративных.
- Все интерактивные элементы должны быть доступны и управляемы с клавиатуры.
- Использовать `:focus-visible` стили с хорошо заметным контрастом.
- Для динамических контролов полностью настраивать `aria-*` (`aria-expanded`, `aria-controls`, labels, states).
- Проверять контраст и не передавать смысл только через фоновые изображения.

## Контраст при отсутствии утверждённого макета

- Если бриф **без финального UI-макета** (нет режима fidelity к Figma/статике по §1.2 `WORKFLOW`), по умолчанию ориентироваться на **WCAG 2.1 уровень AA**: порядка **≥4.5:1** для основного текста и **≥3:1** для крупного текста (ориентир — ≥18 pt / полужирное ≥14 pt) и для **значимых границ/иконок** на смежном фоне.
- Комбинируй **токены палитры** из **`tailwind.config.js`** (**`brand.*`**, **`primary.*`**, **`success|warning|danger`**) поверх **`bg-white`** и других уже заданных поверхностей; не набрасывать «слившиеся» парами пастель/серое на белом без проверки.
- Если пара сомнительна — **проверить** калькулятором контраста (DevTools браузера, WebAIM и т.п.) и **поправить** яркость/оттенок до допустимых значений.
- Сознательно слабый контраст — только при **явном** исключении в задаче по аналогии с waivers в §1.2 `WORKFLOW`.

## Чеклист взаимодействия

- Tab order соответствует визуальной и логической структуре.
- Для modal/offcanvas обеспечено корректное управление фокусом и закрытием.
- Декоративные элементы скрыты от assistive technologies, где это необходимо.
