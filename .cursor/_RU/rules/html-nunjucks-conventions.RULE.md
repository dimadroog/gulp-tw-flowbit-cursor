# Конвенции HTML и Nunjucks

- Использовать Nunjucks loops/macros/includes для повторяющихся паттернов (карточки, списки, сетки).
- Держать шаблонные фрагменты небольшими и предметными (section, card, nav, media).
- Предпочитать data-driven рендеринг вместо копирования HTML-блоков.
- Поддерживать единообразие именования атрибутов и структуры контента в циклах.
- Добавлять краткие комментарии только для исключений и неочевидных ограничений.
- Если у элемента есть и визуальные классы, и JS hook-классы, размещай `js-*` классы в `class`-атрибуте после всех визуальных классов.
- В utility-heavy разметке держи atomic-классы в каноническом порядке `prettier-plugin-tailwindcss`.
- Соблюдай корректную иерархию заголовков (`h1` -> `h2` -> `h3`) без пропусков уровней в структуре документа.
- Рассматривай `section` и `article` как именованные области: внутри каждого должен быть содержательный заголовок.
- Язык страницы по умолчанию русский: используй `<html lang="ru">`, если в задаче явно не указан другой язык.
- Для viewport используй только такое объявление: `<meta name="viewport" content="width=device-width, initial-scale=1">`.
- Если финальный SEO-текст не задан, оставляй заглушки мета-тегов для новых страниц:
  - `<meta name="description" content="description">`
  - `<meta name="keywords" content="keywords">`
- В `<title>` указывай название страницы, совпадающее с именем из реестра страниц (`app/index.html`), если явно не запрошено иное.

## Правила переиспользования

- Если блок встречается более одного раза, выносить его в partial или macro.
- Если меняется только контент, передавать данные вместо клонирования разметки.
