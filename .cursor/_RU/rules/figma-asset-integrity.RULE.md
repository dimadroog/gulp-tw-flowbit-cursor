# Политика целостности Figma-ассетов

- При реализации UI из Figma сохраняй семантику ассетов без подмены: иконки/иллюстрации остаются графикой и не заменяются текстом или эмоджи.
- Для векторной графики вставляй SVG inline напрямую в шаблоны, когда элемент является частью UI-структуры (иконки, логотипы, control-глифы).
- Для растровой графики скачивай файлы в структурированные пути проекта (например `app/img/<page-or-feature>/...`) и в разметке используй только локальные пути.
- Не оставляй во финальной разметке временные внешние URL ассетов (включая Figma MCP asset URLs).
- Не искажать векторную графику: сохраняй исходные `viewBox` и геометрию `path`, масштабирование выполняй через размеры контейнера без деформации.
- Перед завершением валидируй пути к изображениям: не допускаются пустые `src`, неразрешенные относительные пути и битые ссылки в собранном output.
- Если обязательный ассет из дизайна недоступен, останавливай подмену и явно фиксируй проблему как missing asset вместо произвольной замены визуала.
