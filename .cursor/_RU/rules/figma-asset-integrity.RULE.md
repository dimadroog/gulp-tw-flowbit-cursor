# Политика целостности Figma-ассетов

- При реализации UI из Figma сохраняй семантику ассетов без подмены: иконки/иллюстрации остаются графикой и не заменяются текстом или эмоджи.
- Для векторной графики вставляй SVG inline напрямую в шаблоны, когда элемент является частью UI-структуры (иконки, логотипы, control-глифы).
- Файлы SVG в `app/img/` считай **артефактами макета**: после выгрузки из Figma (или MCP) сохраняй их **без несогласованных правок** геометрии (`d`, `path`, `viewBox`, внутренние `transform`) и без произвольной перезаписи корневого `<svg>` до того, как `{% include %}` подтянет файл в разметку. Содержимое включаемого файла должно совпадать с экспортом, если задача явно не разрешает отличие.
- Для растровой графики скачивай файлы в структурированные пути проекта (например `app/img/<page-or-feature>/...`) и в разметке используй только локальные пути.
- Не оставляй во финальной разметке временные внешние URL ассетов (включая Figma MCP asset URLs).
- Не искажай вектор: сохраняй исходные `viewBox` и геометрию `path`; масштаб — только через **внешний контейнер** (CSS/Tailwind на обёртках).
- **Не** задавай `preserveAspectRatio="none"` для UI-иконок/глифов, если дизайн **явно** не требует неравномерного растяжения и это не задокументировано в задаче. Figma часто выставляет этот флаг при растянутом фрейме; предпочитай **повторный экспорт с квадратным фреймом** или правку в макете вместо «ломаной» иконки в вёрстке.
- Избегай несанкционированного пакетного прогона по SVG (SVGO, sed по всем иконкам, автоматическое «нормализовать»), если пользователь или scope задачи это явно не требуют; такие проходы перечисляй в заметках к сдаче с обоснованием.
- Перед завершением валидируй пути к изображениям: не допускаются пустые `src`, неразрешенные относительные пути и битые ссылки в собранном output.
- Если обязательный ассет из дизайна недоступен, останавливай подмену и явно фиксируй проблему как missing asset вместо произвольной замены визуала.
