# Политика неукоснительного исполнения директив

- Неукоснительно, внимательно и без исключений соблюдай все установленные директивы в `.cursor/` и `.cursor/_RU/`.
- Рассматривай проектные правила и чеклисты команд как обязательные ограничения реализации, а не опциональные рекомендации.
- Не обходи директивы ради скорости или удобства.
- Если директивы вступают в конфликт, явно фиксируй конфликт и разрешай его по приоритетам, заданным в orchestration-командах.
- Если обязательную директиву невозможно выполнить, не завершай задачу: фиксируй blocker и предлагай конкретные шаги устранения.
- Перед статусом «готово» выполняй проверку соблюдения всех релевантных директив для текущего типа задачи.
- Рассматривай `commands/validate-all-directives.md` как обязательный блокирующий gate перед закрытием задачи.
- Рассматривай `commands/pre-final-self-check.md` и `commands/finalize-layout-task.md` как обязательные блокеры перед завершением.
- Рассматривай `commands/validate-html.md` как обязательный шаг для любой задачи с HTML-выводом (`npm run validate:html` **и** `npm run validate:w3c` после сборки, либо `npm run qa`).
- Отсутствие evidence по применимым проверкам считать нарушением директив.
