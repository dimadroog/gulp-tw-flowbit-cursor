# Наследование CSS при построении вёрстки

Используйте каскад так, чтобы **потомки получали типографику и цвет от предков**, а не дублировали одни и те же утилиты или блоки `@apply` на каждом вложенном селекторе.

## Базовый уровень проекта (уровень 1)

- **Корень документа:** `html { font-size: var(--font-size); }` в `app/scss/_components.scss` (`@layer base`).
- **Defaults для body:** `body` один раз задаёт `font-sans`, `text-foreground`, `background`, `antialiased` — весь текст на странице наследуется, пока не переопределён.
- **Шкала заголовков:** `h1`–`h4` в `@layer base` меняют только размер/начертание/межстрочный интервал там, где контракт дизайна отличается от body; не объявляйте `font-family` на каждом заголовке, если он совпадает с body.
- **Токены:** цвета и размеры типа — из `app/scss/_variables.scss` и `tailwind.config.js`; предпочитайте наследование + токенные утилиты вместо hex на каждом потомке.

## Что наследуется (задаём выше, переопределяем ниже)

Типичные **наследуемые** свойства — на **ближайшем смысловом предке** (body, обёртка секции, корень BEM-блока, `.card`, `table`, `form`):

- `font-family`, `font-size`, `font-weight`, `font-style`
- `color`, `line-height`, `text-align`
- `letter-spacing`, `word-spacing`, `text-transform`
- `opacity`, `visibility`, `cursor` (если задумано для всего поддерева)

**Переопределяйте только отличие** у потомков (например, у `h2` другой `color`, у ссылки `text-brand`, у активного пункта nav `text-brand`).

## Что не наследуется (всегда явно)

Задавайте на **каждом элементе**, которому они нужны — не полагайтесь на передачу:

- `margin`, `padding`
- `border` (и `border-radius`, если не через общую утилиту на самом элементе)
- `width`, `height`, `min-*`, `max-*`
- `background` / `background-color`
- `display`, `position`, flex/grid-раскладка, `gap`
- `box-shadow`

## Уровни (порядок авторинга)

1. **Глобально** — `body` / `@layer base` в `_components.scss`.
2. **Пояс контента** — `.container` (только ширина/отступы; типографика наследуется от body).
3. **Блок компонента** — корень BEM (например `.site-header`, `.card`): общие `font-family`, `color`, `font-size`, `line-height` **один раз** на блоке, если всё поддерево их разделяет.
4. **Элементы** — заголовки, ссылки, подписи внутри блока: меняйте только отличающееся; не повторяйте наследуемые текстовые свойства.

## Формы и replaced-элементы

- Нативные контролы (`input`, `select`, `textarea`) **не во всех браузерах** наследуют шрифт/размер от родителя — если они должны совпадать с окружающим текстом, используйте **`font-family: inherit`**, **`font-size: inherit`**, **`color: inherit`** (или проектные **`.form-control`** / **`text-base`** по [`tailwind-usage-policy.RULE.md`](tailwind-usage-policy.RULE.md)).
- Не полагайтесь на наследование для **focus ring**, **границ** и **padding** у контролов.

## `inherit` и `initial`

- **`inherit`** — потомок должен явно совпасть с родителем (например, цвет ссылки внутри тонированной карточки по макету).
- **`initial`** — сброс к UA только при баге из-за наследования; краткий комментарий в SCSS.

## Tailwind / `@apply` в этом репозитории

- Предпочитайте **обычные списки селекторов** и семантические классы вместо `:where()` / `:is()`, если нет задокументированного исключения по каскаду — см. [`css-selector-pseudo-classes.RULE.md`](css-selector-pseudo-classes.RULE.md).
- Предпочитайте **один** текстовый `@apply` на **родительском** семантическом классе; у потомков — модификаторы (`.is-active`, `.site-header-logo-dot`) для отличий.
- Не дублируйте одинаковые `text-14 text-neutral-800` (и эквиваленты) на каждом вложенном `a`/`span`, если блок-предок уже задал те же значения.
- Утилиты в Nunjucks — для **раскладки и исключений**, а не для повторного объявления наследуемой типографики на каждом узле.

## Проверка (перед завершением секции)

- [ ] Базовая типографика на `body` / корне блока, а не продублирована на трёх+ вложенных селекторах без причины по дизайну.
- [ ] Отступы и границы заданы на элементах, которым принадлежит box, а не предполагаются от родителей.
- [ ] Новый SCSS не повторяет `font-family` / `font-size` / `color` / `line-height` у детей, уже наследующих те же значения от предка в том же блоке.
- [ ] Поля формы используют `.form-control` / задокументированные утилиты или явный `inherit`, где нужно совпадение с текстом родителя.
