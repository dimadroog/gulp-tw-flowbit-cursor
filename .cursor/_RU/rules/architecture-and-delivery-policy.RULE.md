# Политика архитектуры и доставки

- Во всех решениях по реализации приоритетом ставь скорость загрузки страниц и runtime-производительность.
- Используй best practices и рекомендации Lighthouse/PageSpeed как базовый ориентир для performance-решений.
- Работай в рамках текущей архитектуры проекта и используемого инструментария.
- Для каждого изображения явно задавай `loading="lazy"` или `loading="eager"` по контексту.
- Оптимизируй все изображения для веба и держи исходные размеры максимально близкими к фактическим отображаемым размерам.
- Используй `picture`/`srcset`, чтобы отдавать наиболее подходящее изображение для текущего viewport и плотности экрана.
- Используй `WOFF`/`WOFF2` как базовые форматы webfont для поддержки современных браузеров.
- Предпочитай self-hosted подключение шрифтов и скриптов с сервера проекта (кроме проектов только на системных шрифтах).
- Подключай только реально используемые семейства и начертания шрифтов.
- Для кастомных шрифтов задавай `font-display: swap`, чтобы не блокировать первичный рендер текста.
- Предпочитай композицию через partials/includes/macros вместо дублирования разметки.
- Не поставляй лишние JS/CSS и тяжелые медиа, если есть эквивалентные более легкие варианты.
- Делай верстку контентоустойчивой: проверяй поведение при удлинении текста и при замене изображений на другие размеры/пропорции.
