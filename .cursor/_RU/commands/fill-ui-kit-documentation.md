# fill-ui-kit-documentation

Изучи `app/ui-kit.html` и поэтапно наполни проектную документацию по разделам ниже.

## Source of truth

- Основной источник: `app/ui-kit.html`
- Дополнительная проверка сборки: `dist/ui-kit.html`

## Задача

Собери структурированную документацию для каждого раздела:

1. Краткое назначение раздела
2. Используемые UI-элементы и классы
3. Примеры использования (минимум один)
4. Ограничения, состояния и заметки по accessibility

## Разделы

1. Цветовая палитра
2. Система отступов
3. Типографика:
   - Шрифты
     - Семейства (акцидентный, основной)
     - Размеры
     - Начертания
   - Заголовки (`h1`-`h6`)
   - Абзацы
   - Списки (нумерованные, маркированные)
   - Цитаты (`blockquote`)
   - Таблицы
4. Ссылки
5. Кнопки
6. Элементы ввода:
   - Поля
   - Выпадающие списки
   - Чекбоксы
   - Радио-переключатели
   - `range-sliders`
7. Пропорции сторон изображений

## Правила заполнения

- Описывай только данные, подтвержденные в `ui-kit.html`.
- Если раздел представлен частично, добавляй `TODO` и перечисляй недостающие данные.
- Для таблиц явно укажи, что пример в текущем `ui-kit.html` закомментирован, и добавь `TODO` на активный пример.
- Для шрифтов не придумывай значения: если в `ui-kit.html` нет токенов/семейств, добавляй `TODO` и ссылку на источник (`scss` или design tokens).
- Для каждого интерактивного элемента добавляй краткую a11y-заметку (label, focus, `aria`, состояния).

## Ожидаемый результат

- Один markdown-документ с оглавлением по всем разделам.
- Для каждого раздела указан статус:
  - `complete` - полностью задокументирован по `ui-kit.html`
  - `partial` - частично задокументирован, есть явные `TODO`
  - `missing` - в `ui-kit.html` нет данных

## Критерии приемки

- Все 7 разделов присутствуют.
- В каждом разделе есть минимум один пример.
- Все пробелы данных отмечены как `TODO` без догадок.
