# Критерии готовности: Cursor Infrastructure Iteration 1

Используй этот список для проверки запуска инфраструктуры только в IDE.

> **Заметка:** это **исторический / smoke-test** список для первой волны `.cursor/`. **Фактическое завершение** верстки определяется `commands/run-layout-task.md`, `finalize-layout-task.md`, `validate-all-directives.md` и **`npm run qa`** (см. `commands/validate-html.md`). Прохождение только этого списка **не** равно прохождению Nu или полного QA проекта.

## Обязательные результаты

- Общие правила присутствуют и последовательно применяют политики стека, доступности, стилей и JS.
- Общие правила и команды явно закрепляют приоритет скорости загрузки страниц и performance.
- Есть команды для каркаса страницы, сборки секций, framework-рефакторинга и a11y-проверки.
- Есть skills для генерации страниц, шаблонизации Nunjucks, сборки секций на Tailwind и аудитов интерактивов.
- Топология агентов и правила оркестрации задокументированы и применимы на практике.

## Чеклист верификации

- [ ] Новую страницу можно собрать через `new-page` и привести к baseline-политике.
- [ ] В интерактивной секции есть явный выход decision gate: framework или custom JS.
- [ ] Повторяющиеся шаблонные блоки реализованы через loops/includes/macros.
- [ ] Результат accessibility-review явно покрывает ARIA и focus-поведение.
- [ ] BEM-исключения обоснованы и ограничены по объему.
- [ ] `performance-checklist` применяется при сборке страниц/секций, а ключевые оптимизации зафиксированы.
- [ ] Контентоустойчивость подтверждена на длинном тексте и замене изображений с разными пропорциями.
- [ ] Соблюдена политика доставки шрифтов (`WOFF/WOFF2`, self-hosted где применимо, только используемые начертания, `font-display: swap`).
